# Delayed Notifier

Delayed Notifier — это сервис отложенных уведомлений, который позволяет создавать уведомления с заданной датой и временем отправки и автоматически доставлять их через различные каналы (Email, Telegram).

Сервис принимает запросы на создание уведомлений, сохраняет их состояние, отправляет в очередь (RabbitMQ), а затем — в нужное время — отправляет сообщение. Также поддерживается отмена уведомлений и проверка их текущего статуса.

---

## Возможности

- создание уведомлений с отложенной отправкой;
- получение текущего статуса уведомления;
- отмена запланированного уведомления;
- фоновая обработка очереди уведомлений;
- повторная отправка при ошибках;
- поддержка нескольких каналов доставки (Email, Telegram);
- хранение и быстрая проверка статусов через Redis;
- простой UI для ручного тестирования.

---

## API

Сервис предоставляет HTTP API:

- `POST /notify` — создать уведомление с датой и временем отправки;
- `GET /notify/{id}` — получить статус уведомления;
- `DELETE /notify/{id}` — отменить уведомление.

---

## Архитектура

Основные компоненты проекта:

- **HTTP API** — принимает запросы на создание, получение и отмену уведомлений;
- **RabbitMQ** — очередь для фоновой обработки уведомлений;
- **Consumer** — обрабатывает сообщения из очереди и отправляет уведомления в нужное время;
- **Redis** — хранение и быстрый доступ к статусам уведомлений;
- **Senders** — отправка уведомлений через разные каналы (Email, Telegram);
- **Frontend** — простой HTML + JS интерфейс для тестирования.
