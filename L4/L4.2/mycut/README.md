# Distributed Cut CLI Tool (mycut)

## Описание
`mycut` — это распределённая реализация утилиты `cut` с поддержкой многосерверного режима и кворума.

Утилита позволяет:
- Работать с колонками файлов через флаги `-f`, `-d`, `-s`.
- Запускать локально или распределённо через несколько серверов.
- Достигать кворума N/2+1 для подтверждения успешной обработки.
- Использовать параллельные горутины и каналы для оптимизации.


## Установка
Скомпилируйте бинарник:

```bash
go build -o mycut
```

После сборки появится файл `mycut`, который можно запускать.


## Использование

### Локальный режим (как обычный cut)

```bash
cat new_data.txt | ./mycut -f 1,3
```
- `-f` — номера колонок
- `-d` — разделитель (по умолчанию `\t`)
- `-s` — выводить только строки с разделителем

Пример:
```bash
cat new_data.txt | ./mycut -f 1,3
```
Вывод:
```
name    city
Alice   Riga
Bob     Tallinn
Charlie Vilnius
```


### Серверный режим

Запускаем сервер на нужном порту:

```bash
./mycut --server --port 9001
./mycut --server --port 9002
./mycut --server --port 9003
```
Каждое соединение обрабатывается параллельно.


### Распределённый режим

Клиентский запуск с несколькими серверами:

```bash
cat new_data.txt | ./mycut -f 1,3 --nodes localhost:9001,localhost:9002,localhost:9003
```

Вывод будет включать строки, обработанные серверами, после кворума:
```
Quorum reached
name    city
Alice   Riga
Bob     Tallinn
Charlie Vilnius
```


## Сравнение с обычным cut

### Пример теста на времени выполнения:
```bash
# Локальный cut
time cut -f 1,3 new_data.txt

# Распределённый mycut
time cat new_data.txt | ./mycut -f 1,3 --nodes localhost:9001,localhost:9002,localhost:9003
```
- На маленьких файлах `cut` быстрее из-за отсутствия сетевых накладных расходов.
- На больших данных или нескольких физических машинах `mycut` масштабируется и показывает преимущество.


## Структура проекта

- `client.go` — клиентская логика, деление данных и сбор результатов.
- `cut.go` — обработка строк и выбор колонок.
- `main.go` — запуск локального или распределённого режима.
- `quorom.go` — проверка кворума N/2+1.
- `server.go` — сервер, который принимает строки, обрабатывает их и возвращает результат.
- `data.txt`, `new_data.txt`, `big.txt` — примеры файлов для тестирования.


## Примечания

- Для распределённого режима обязательно запускать серверы на разных портах.
- Кворум достигается, когда N/2+1 серверов успешно обработали свой кусок.
- Клиент собирает результаты всех серверов и выводит их в stdout после кворума.
- Утилита полностью использует параллелизм через goroutines и каналы.
